This document contains the user personas that we consider when developing
Projectify

# Lean

- Early 30s
- Works as a freelance internet marketer
- Uses project management apps to coordinate ad campaigns, either in teams or
  alone
- Uses tasks to point to documentation so that team members can look at tasks
and know what to do next
- UI shortcuts and templates are a *must* as managing ad campaigns is repetitive
- Projects can have over 50 steps
- Prefers apps that feel snappy
- "A UX is non-negotiable"
- Jira has too many features. Has not used "Rovo" in Jira before.

# Kasama

- Early 30s
- Works as a full stack developer for a SaaS company
- Uses Jira and has daily scrum meetings.
- Requirements for a project management app:
  - "I want to see what my tasks are"
  - "I want to share my task status with others, especially product management"
- Largely, two ticket types on Jira:
  - "Do soon"
  - "Do later" (backlog)
- Rarely looks at other people's tasks
- Doesn't fully understand Jira and prefers to not use it too much.
- Jira pain points:
  - Too many buttons in UI
  - Creating new tasks is annoyingly difficult. Tasks that she creates
    unintentionally become "stories"
  - Tasks have too many fields.
  - Task start dates are hard to find.
- Uses Jira 1-2 times per day.
- Doesn't bookmark Jira ticket URLs
- People don't leave comments on Jira tickets
- Jira integrates with GitHub, so tickets become *done* when a GitHub pull
  request merges.

# User interviews

Here are questions that you can ask a user when testing Projectify together
with them:

* Are you confident that you can find your next task on the dashboard?
* Do you feel adding new tasks is easy?
* Do you feel updating your tasks after you've created them is easy?
* Can you easily categorize tasks on Projectify?
* Do you think that information about tasks on Projectify is easy to share
with other team members?
* Do you confidently understand how Projectify works?
* Does Projectify show you only relevant tasks and other information?

Ask the tester to answer with a number between 1 and 10 for each question.
1 means "I fully disagree with this statement" and 10 means "I fully
agree with this statement."
